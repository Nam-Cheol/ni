# Risks and security

## RISK-001: model self-approval

Severity: high

Mitigation: readiness and lock state are CLI-enforced. The model cannot declare completion alone.

## RISK-002: criteria weakening

Severity: high

Mitigation: accepted evaluations and non-goals must be preserved after lock.

## RISK-003: Codex lock-in

Severity: medium

Mitigation: Codex skills are UX. Kernel concepts must remain provider-neutral.
