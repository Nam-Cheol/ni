# Project brief

## Purpose

`ni` compiles ongoing planning conversations into a locked project contract, then uses that contract to derive a project-specific harness.

## Problem

Strict SPEC-first harnesses can become too heavy, too linear, and too tied to one execution environment. `ni` moves the control boundary to the planning layer.

## Success definition

A user can maintain planning docs, validate readiness, lock the plan, and generate a 4000-character-or-less goal prompt before implementation begins.
