# Actors and outcomes

## Actors

- User: creates or revises the project plan.
- Planning model: helps draft docs and detect gaps.
- CLI: validates readiness and lock state.
- Execution agent: later derives a project-specific harness from the locked contract.

## Outcomes

- The user has a project contract that is explicit enough to execute.
- The model cannot silently decide that incomplete docs are complete.
- Execution starts from a locked contract, not a drifting conversation.
