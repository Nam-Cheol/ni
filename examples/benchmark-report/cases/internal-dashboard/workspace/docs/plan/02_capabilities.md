# Capabilities

## CAP-001: Capture the vague dashboard request as planning state

Record the original dashboard request, visible assumptions, non-goals, risks,
and blocker questions without turning it into implementation instructions.

## CAP-002: Block readiness while dashboard intent is unresolved

Keep readiness blocked until the user confirms the primary actor, attention
definition, allowed account signals, source systems, privacy and access-control
constraints, and planning-meeting acceptance evidence.
