# Capabilities

## CAP-001: Capture dashboard intent without starting implementation

Record the vague dashboard request as project purpose, accepted requirements,
risks, artifacts, assumptions, and blocker questions. Do not turn the request
into implementation instructions yet.

## CAP-002: Block readiness while workflow and data-source questions remain open

Keep readiness blocked until the user resolves who the dashboard serves, which
workflow comes first, which data source is allowed, and what customer-data
boundary applies.
