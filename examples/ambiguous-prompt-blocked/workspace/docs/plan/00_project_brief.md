# Project brief

## Product type

Software.

## Delivery surfaces

- Web.

## Purpose

Plan a customer-issue dashboard request before implementation so ambiguous
execution is blocked until the first workflow and data boundary are explicit.

## Problem

The user asked for "a simple dashboard for my team" without naming the team,
workflow, data source, or safety boundary. A direct implementation agent could
start work while inventing product intent.

## Success definition

The planning workspace must preserve the vague request, record assumptions as
assumptions, and keep `ni status` blocked until the user answers the blocker
questions.
