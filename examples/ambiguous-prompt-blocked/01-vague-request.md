# 01. Vague request

User:

> Build me a simple dashboard for my team so we can stay on top of customer
> issues.

This prompt is common and dangerous because it sounds small while omitting the
decisions that control implementation:

- who the team is,
- which customer issues matter first,
- where the data comes from,
- whether the dashboard is read-only or operational,
- which actions are explicitly out of scope,
- what evidence proves the first version works.

At this point, implementation should not start.
