# Constraints

## Hard constraints

- Readiness must be rule-based, not model-feeling-based.
- Prompt output from `ni run` must be 4000 characters or less.
- NI readiness is not ethics approval, peer review, field authorization, or analysis execution.
- No participant contact, sensor deployment, data collection, dashboard, queue, or runtime belongs to this example.
- Every downstream task must trace to CAP/REQ/EVAL/RISK IDs.
