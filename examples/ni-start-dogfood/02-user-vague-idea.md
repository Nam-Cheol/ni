# 02. user invokes ni-start

## User

Invoke `ni-start` on the fresh workspace.

## What is known

- `ni init` created the planning workspace.
- The user wants model-guided planning, not manual contract editing.
- User intent: planning, not implementation.

## What is still ambiguous

- What reality the project should change.
- Who the primary actors are and which outcomes they need.
- Which delivery surface the plan should target.

The empty workspace is intentionally not sent to an implementation agent. NI
keeps the work in planning until the readiness gate says otherwise.
